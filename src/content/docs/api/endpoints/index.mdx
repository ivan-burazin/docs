# API Endpoints

The Daytona API provides several endpoints grouped by resource type. Each endpoint follows REST conventions and requires authentication via API key.

## Available Resources

### API Keys
Manage API authentication keys for accessing the Daytona API. [Learn more](/api/endpoints/apikeys)

### Builds
Control and monitor build processes for your workspaces. [Learn more](/api/endpoints/builds)

### Git Providers
Configure and manage Git provider integrations. [Learn more](/api/endpoints/git)

### Container Registry
Handle container registry operations and credentials. [Learn more](/api/endpoints/registry)

## Common Parameters

Many endpoints support these common query parameters:

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | integer | Page number for paginated results |
| `per_page` | integer | Number of items per page (max: 100) |

## Response Format

All endpoints return responses in a consistent JSON format:

```json
{
  "data": {
    // Resource-specific data
  },
  "meta": {
    "timestamp": "2024-01-20T12:00:00Z"
  }
}
```

For detailed information about specific endpoints, select a resource category from the navigation menu.
